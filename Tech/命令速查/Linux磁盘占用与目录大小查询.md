## 🤖 Assistant

作为一名工程师，在 Linux 环境下排查磁盘空间问题时，我通常会遵循“从宏观到微观”的原则。以下是常用的命令及其实战用法：

### 1. 查看整体磁盘占用 (宏观)
首先，我们需要知道哪个挂载点（分区）满了。

*   **命令：`df -h`**
    *   `-h` (human-readable)：以 GB、MB 等易读格式显示。
    *   **关注点：** 查看 `Use%`（使用率）和 `Mounted on`（挂载点）。

```bash
df -h
```

---

### 2. 查看特定目录的大小 (微观)
确定了哪个分区满了以后（比如 `/var` 或 `/home`），使用 `du` 命令来定位大文件。

*   **常用命令：`du -sh [目录名]`**
    *   `-s` (summary)：仅显示总计。
    *   `-h` (human-readable)：易读格式。

```bash
# 查看当前目录下每个文件夹的总大小
du -sh *

# 查看指定目录（如 /var）的总大小
du -sh /var
```

*   **进阶：查看目录层级大小**
    如果你想看某个线下多级目录的大小，但不希望被成千上万的文件刷屏，可以使用 `--max-depth`：

```bash
# 只查看当前目录下一级子目录的大小
du -h --max-depth=1 /path/to/directory
```

---

### 3. 排序：寻找“空间杀手”
通常我们需要找出占用空间最大的前几个目录：

*   **组合命令：**
```bash
# 查看当前目录下前 10 个最大的文件或目录
du -ah . | sort -rh | head -n 10
```
*   参数解释：
    *   `du -ah`: `-a` 包含文件，`-h` 易读格式。
    *   `sort -rh`: `-r` 反向排序（大的在前），`-h` 识别人类可读符号（如 1G > 2M）。
    *   `head -n 10`: 取前 10 行。

---

### 4. 工程师推荐的交互式工具 (ncdu)
如果你有权限安装软件，`ncdu` 是最高效的选择。它提供了一个基于终端的 UI，可以像文件管理器一样浏览并删除文件。

*   **安装：**
    ```bash
    sudo apt install ncdu  # Debian/Ubuntu
    sudo yum install ncdu  # CentOS/RHEL
    ```
*   **使用：**
    ```bash
    ncdu /
    ```
    *   使用方向键导航，`d` 键直接删除文件，`q` 退出。

---

### 5. 工程师的避坑指南 (Tips)
在处理磁盘占用时，有几个常见的坑需要注意：

1.  **文件已删除但空间未释放：**
    如果一个进程正在写入大日志文件，你直接 `rm` 删除了它，磁盘空间可能不会立即释放（因为文件句柄还被进程持有）。
    *   **排查：** `lsof | grep deleted`
    *   **解决：** 重启对应服务，或者通过 `> file.log` 清空文件内容而非直接删除。

2.  **隐藏文件：**
    `du -sh *` 默认不会显示以 `.` 开头的隐藏目录（如 `.cache`）。
    *   **解决：** 使用 `du -sh .[^.]* *` 或 `ncdu`。

3.  **Inode 耗尽：**
    有时候 `df -h` 显示还有空间，但无法创建文件。这可能是 Inode 用完了（大量小文件导致）。
    *   **查看：** `df -i`

建议先从 `df -h` 开始，定位到大分区后，用 `du -sh * | sort -hr` 逐级向下排查。
