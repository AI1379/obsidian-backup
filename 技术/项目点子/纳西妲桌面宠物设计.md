# 🌿 纳西妲桌面宠物 — 设计方案

> **项目状态**: 概念设计阶段
> **创建日期**: 2026-05-22
> **技术栈**: Tauri (Rust + WebView) + Live2D Cubism Web SDK

---

## 1. 项目概述

一个轻量级桌面宠物应用。纳西妲以 Live2D 形态常驻桌面后台，收到消息时从屏幕侧边弹出提醒，用户点击后打开聊天窗口与 AI 进行对话。

### 核心目标

| 目标 | 说明 |
|------|------|
| **轻量常驻** | 后台资源消耗极低，不影响日常使用 |
| **消息感知** | 实时感知来自各 channel 的消息，主动通知用户 |
| **便捷对话** | 点击即可开聊，作为独立 channel 接入 Gateway |
| **表现力** | Live2D 驱动表情、口型、动作，赋予角色生命力 |

---

## 2. 技术选型

### 2.1 客户端框架：Tauri

| 考量 | 选择 | 理由 |
|------|------|------|
| 框架 | **Tauri 2.x** | Rust 后端 + 系统 WebView，内存 ~20MB（Electron 约 100MB+） |
| 前端 | **Vanilla TS / 轻量框架** | Live2D 渲染 + 简单 UI，不需要重型框架 |
| 渲染 | **Live2D Cubism Web SDK** (WebGL) | 官方 Web 方案，GPU 渲染，CPU 占用极低 |
| 通信 | **WebSocket** | 与 Gateway 实时双向通信 |

**为什么不是 Electron？**
打包体积 ~150MB vs Tauri ~5MB，内存差 5-10 倍。桌宠需要常驻后台，轻量是第一优先级。

**为什么不是原生应用？**
开发成本高，Live2D Native SDK 接入复杂。Web SDK 成熟且在 WebView 中性能足够。

### 2.2 窗口形态

应用包含两个窗口（Tauri 支持多窗口）：

| 窗口 | 用途 | 尺寸 | 特性 |
|------|------|------|------|
| **宠物窗口** | Live2D 渲染 + 交互 | ~200×300px | 无边框、透明背景、置顶、可拖拽、部分区域点击穿透 |
| **聊天窗口** | 消息列表 + 输入框 | ~400×600px | 无边框/圆角、置顶、可拖拽 |

### 3.2 通信协议

客户端作为一个独立 channel 接入 Gateway，通过 WebSocket 通信：

```typescript
// 客户端 → Gateway
interface ClientMessage {
  type: "chat_message" | "chat_open" | "chat_close" | "ping";
  content?: string;       // 聊天内容
  session_id?: string;    // 会话标识
}

// Gateway → 客户端
interface ServerMessage {
  type: "notification" | "chat_response" | "expression" | "pong";
  content?: string;       // 文本内容
  source?: string;        // 消息来源 channel (qq/cron/desktop)
  expression?: string;    // 表情指令 (happy/think/wave...)
  actions?: string[];     // 动作指令
}
```

### 3.3 状态机

```
                    ┌──────────┐
          启动      │  托盘模式  │ ← 最小资源
         ────────→  │ (隐藏窗口) │
                    └─────┬────┘
                          │ 收到消息 / 用户唤醒
                          ↓
                    ┌──────────┐
                    │  弹出模式  │ ← Live2D 低帧率 (10fps)
                    │ (侧边宠物) │
                    └─────┬────┘
                     ↕    │ 点击
                  超时/    ↓
                  忽略  ┌──────────┐
                    ←── │  聊天模式  │ ← Live2D 正常帧率 (30fps)
                        │ (展开窗口) │
                        └─────┬────┘
                              │ 关闭
                              ↓
                        回到弹出/托盘
```

## 4. Live2D 渲染层

### 4.1 模型需求

需要一个纳西妲 Live2D 模型（Cubism 格式，.moc3 + 纹理 + 动作文件）。模型应包含：

- **基础表情**: 默认、开心、思考、惊讶、难过、困倦
- **基础动作**: 待机呼吸、点头、摇头、挥手、伸懒腰
- **口型参数**: `ParamMouthOpenY`、`ParamMouthForm`

### 4.2 表情驱动方案

AI 输出文本时嵌入标记，客户端解析后驱动表情：

```
#happy 旅行者你好呀！今天过得怎么样？
#think 嗯...让我想想这个问题...
#wave 我先去忙啦，回头见！
```

| 标记 | 映射 | 说明 |
|------|------|------|
| `#happy` | 笑脸 + 轻微点头 | 开心 |
| `#think` | 歪头 + 眼睛上瞥 | 思考 |
| `#surprise` | 眼睛放大 + 后仰 | 惊讶 |
| `#sad` | 垂眼 + 低头 | 难过 |
| `#wave` | 挥手动作 | 打招呼/告别 |
| `#shy` | 脸红 + 低头 | 害羞 |
| （无标记）| 默认表情 | 正常对话 |

### 4.3 口型同步

**MVP 阶段：文本驱动口型**

```typescript
// 按字符映射嘴巴开合
// 每个字符持续约 80-120ms（模拟说话节奏）
function textToLipSync(text: string): LipFrame[] {
  const frames: LipFrame[] = [];
  for (const char of text) {
    if (/[aeo啊哦额哎]/.test(char)) {
      frames.push({ open: 0.9, form: 0.3, duration: 100 }); // 大开口
    } else if (/[iu一三五六七]/.test(char)) {
      frames.push({ open: 0.4, form: 0.8, duration: 90 }); // 小开口
    } else if (/[，。！？、]/.test(char)) {
      frames.push({ open: 0.0, form: 0.0, duration: 200 }); // 停顿
    } else {
      frames.push({ open: 0.6, form: 0.5, duration: 80 }); // 默认
    }
  }
  return frames;
}
```

**后续可升级：** 接入 TTS → 音频驱动口型，精度更高。

### 4.4 资源控制策略

| 状态 | 帧率 | GPU | 说明 |
|------|------|-----|------|
| 托盘模式 | 0 fps | 无 | 窗口隐藏 |
| 弹出模式 | 10 fps | 低 | 仅待机呼吸+眨眼 |
| 聊天模式 (聆听) | 15 fps | 低 | 微动 |
| 聊天模式 (说话) | 30 fps | 中 | 口型+表情驱动 |
| 聊天模式 (空闲) | 10 fps | 低 | 降帧省资源 |

## 5. 交互设计

### 5.1 弹出行为

收到消息时：
1. 从屏幕右下角/右侧滑出宠物窗口（~200×300px）
2. Live2D 播放"注意到"表情（抬头、眼睛放大）
3. 消息来源显示为气泡提示（不展开完整内容）
4. 15 秒无交互自动收起

弹出方向可配置：右下角（默认）、右侧、右下角偏上。

### 5.2 聊天窗口

点击宠物后：
1. 宠物窗口平滑展开为聊天窗口（或从宠物位置弹出聊天窗口）
2. 上方：Live2D 模型（缩小为头像尺寸）+ 表情驱动
3. 下方：聊天气泡列表 + 输入框
4. 支持 Enter 发送，Shift+Enter 换行
5. Esc 或关闭按钮收起

### 5.3 宠物交互（彩蛋）

| 交互 | 反应 |
|------|------|
| 点击头部 | "诶？别戳我啦~" |
| 拖拽 | 跟随移动，松手后回弹动画 |
| 长时间不互动 | 偶尔打哈欠 / 翻书 / 看风景 |
| 双击 | 切换待机动作 |

### 5.4 系统托盘菜单

```
纳西妲 🌿
├─ 显示/隐藏
├─ 弹出位置 → 右下角 / 右侧 / 自定义
├─ 消息通知 → 开/关
├─ 帧率设置 → 省电 / 平衡 / 流畅
├─ 关于
└─ 退出
```

## 6. 本地存储

| 数据 | 存储方式 | 说明 |
|------|----------|------|
| 聊天历史 | SQLite (Tauri 插件) | 本地缓存，减轻后端负担 |

## 8. 风险与注意事项

| 风险 | 缓解措施 |
|------|----------|
| Live2D 模型版权 | 使用官方资源或自制，确认使用许可 |
| Tauri WebView 兼容性 | WebView2 在 Windows 10+ 内置，兼容性良好 |
| WebSocket 断线 | 自动重连 + 指数退避 + 离线缓存 |
| 多窗口动画卡顿 | 弹出动画用 CSS transition，避免 JS 逐帧计算 |
| 全屏游戏干扰 | 检测全屏应用时自动降为托盘模式 |
