
# 基本设计
## 1. 项目概况

- **核心理念**：将古典乐理（和声、调性、节奏）解构为核心战斗逻辑，而非传统的下落式音游。
    
- **游戏类型**：2D 横板探索 + 指令式卡牌/回合制 RPG。
    
- **美术风格**：类《重返未来：1999》的欧洲中世纪/近代风，结合神秘学与古典乐美学。
    
- **核心情绪**：优雅、忧郁、浪漫主义、疗愈与补全。

## 2. 世界观与剧情 (Narrative)

- **世界背景**：世界是一首停滞在“休止符”上的未完交响曲。风雪是信号缺失的“白噪音”，极光是天际飘荡的“磁带”。
    
- **主角设定**：**留声机/收音机化形（寻频者）**。唯一能记录并播放外界信号的存在，负责接续断掉的旋律。他可以有名字，也可以让玩家取名。比如说把拉赫的名字倒过来，Sergei变成Iegres这样，或者把Rachmaninov反过来Voninamhcar然后再转写一下。
    
- **核心剧情线**：
	1. **开端**：雪国，列车，阴雨天，窗上全是雨雾。这里可以很意识流地描绘这个场景：车里不展现内饰，只有简笔勾勒的座位，主角坐在车上，旁边只有一个方框是窗，窗上雾蒙蒙的，隐约可见外面的世界。这里没有背景音乐，只有白噪音。
	2. **初见这个世界**：花园，清泉，流水，小屋，但是仍然笼罩在雾气之中，营造那种阴郁的整体的氛围。这里的氛围会有点像山中好长日初见彩特琳德登上空之神殿时剧情的色彩，但是需要仔细斟酌表达的形式。这里是初见这个世界，游戏上可以介绍基础设定，然后可以有一些音乐小妖精作为初始的怪物。这里的背景可以是肖邦夜曲，但是也许需要做一些改编。
	3. **剧情发展**：黑森林，原始，阴森，有些令人恐惧。可以自己写点中古调式的曲子配乐。
	4. **逐渐进入高潮**：湖中尖塔，月光，月光会不断地变化，对应游戏内部战斗环境变化。这里的怪物暂且还是音乐相关的妖精。月光变换，对应战斗环境的调式变换。这里可以引入一些欧洲的神话，月亮在欧洲神话中常有与音乐有关的意向。场景有点水仙十字格式塔和深境螺旋的感觉。背景可以放月光奏鸣曲第一乐章。
	5. **塔尖**：第一个boss战，战斗曲可以在李斯特钟的基础上改，boss本身也可以加入一些钟表元素。战斗结束后月光重新出现，但是整个场景只有朦胧的月光和被雾笼罩的夜空。主角将在这里完成一次世界观的展开：世界是倒悬的，而水面才是真实的星空。主角从塔上跳下，进入世界的反面。这里也是剧情的一个高点，象征从灰暗阴沉的物理世界走进纠结矛盾的内心世界。
	6. **倒置的塔**：向下离开尖塔，这时候内部景观需要有很大的变化。妖精消失，怪物变成调律不齐的乐器，背景音乐开始变得紧张，比如从月光一变成月光三。
	7. **来时路的重放**：倒着走一遍来时的路，但是怪物和环境都会有一些变化，整个氛围逐渐变得紧张和纠结，也可以增加一些高难的战斗。这里的战斗和探索也许可以放一些无调性的现代主义的音乐。
	8. **废弃车站**：第二个boss战，也是终局之战。可以继续参考山中好长日和水仙十字的写法。这一战的敌人象征内心的病魔和抑郁。战斗结束乌云散去，极光亮起。
	9. **结尾**：伴随拉二的音乐，列车启程。天空逐渐亮起，然后黎明到来，拨云见日。

## 3. 战斗系统 (Combat System)

核心机制借鉴“元素反应”，命名为“和声共鸣 (Harmony Resonance)”。

### 3.1 基础属性：调性 (Tonality)

- **大调 (Major)**：代表光明、秩序、爆发。
    
- **小调 (Minor)**：代表忧郁、混沌、持续。
    
- **克制关系**：大调与小调互为攻守加成（类光暗属性）。

### 3.2 核心玩法：和声共鸣

角色技能会给敌人挂上“音名标记 (C, D, E...)”，不同音名的组合触发不同反应：

- **大三和弦 (Major Triad)**：高额爆发伤害 + 己方攻击 Buff。
    
- **小三和弦 (Minor Triad)**：持续负面伤害 (DoT) + 降低敌人速度。
    
- **不协和音 (Dissonance)**：触发高风险的“真实伤害”，但自身可能受到反噬。真伤穿防，但是烧自身的血。因为没有设计奶妈，所以这里数值需要仔细斟酌。

### 3.3 角色职能分配 (Classes)

根据乐器发声原理分为四大类职业：

1. **弦乐 (Strings)**：多段攻击、群体伤害。负责铺设基础音符标记（主输出/辅助输出）。
2. **管乐 (Winds)**：单体高穿透、高爆发。负责精准击破或触发关键共鸣（刺客/狙击手）。
3. **打击乐 (Percussion)**：不参与凑和弦，负责**节奏控制 (Tempo)**。
    - **机制**：引爆场上和弦、拉条/推条、破坏敌方护盾（节奏稳定性）。
4. **钢琴 (Piano - 乐器之王)**：
    - **机制**：全能位。可补全缺失的任何音符；拥有“黑白键”双形态切换；通过“场地魔法”改变全场调性。
5. **留声机 / 主角：** 记录，复播，未必有战斗机制。

## 4. 技术选型 (Technical Specs)

- **开发引擎**：**Unity**。
    
    - **理由**：对 Live2D/Spine 的 2D 骨骼动画支持最成熟，适合做高质量立绘交互；UI 系统（UGUI/UI Toolkit）适合重度 UI 的卡牌 RPG。
        
- **音频处理**：推荐引入 **FMOD** 或 **Wwise** 中间件，用于处理复杂的动态音轨叠层和实时调性切换。
    
- **技术挑战与对策**：
    
    - **UI 性能**：采用 Canvas 动静分离。
        
    - **资源管理**：使用 Addressables 实现音频与美术资源的异步加载。
        
    - **内存优化**：针对频繁生成的音符/特效使用对象池 (Object Pool)。

## 5. 开发路线 (Roadmap)

1. **Demo 阶段**：实现钢琴 + 弦乐 + 鼓的战斗原型，验证“和声共鸣”的可玩性。
    
2. **垂直切片**：完成“雪国列车”场景表现及一个完整的剧情小节。
    
3. **美术迭代**：确立“乐器娘”的拟人化程度（建议保留乐器特征的优雅风格）。
