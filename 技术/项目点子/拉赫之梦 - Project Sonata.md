
# 基本设计
## 1. 项目概况

- **核心理念**：将古典乐理（和声、调性、节奏）解构为核心战斗逻辑，而非传统的下落式音游。
    
- **游戏类型**：2D 横板探索 + 指令式卡牌/回合制 RPG。
    
- **美术风格**：类《重返未来：1999》的欧洲中世纪/近代风，结合神秘学与古典乐美学。
    
- **核心情绪**：优雅、忧郁、浪漫主义、疗愈与补全。

## 2. 世界观与剧情 (Narrative)

- **世界背景**：世界是一首停滞在“休止符”上的未完交响曲。风雪是信号缺失的“白噪音”，极光是天际飘荡的“磁带”。
    
- **主角设定**：**留声机/收音机化形（寻频者）**。唯一能记录并播放外界信号的存在，负责接续断掉的旋律。他可以有名字，也可以让玩家取名。比如说把拉赫的名字倒过来，Sergei 变成 Iegres 这样，或者把 Rachmaninov 反过来 Voninamhcar 然后再转写一下。
    
- **核心剧情线**：
	1. **开端**：雪国，列车，阴雨天，窗上全是雨雾。这里可以很意识流地描绘这个场景：车里不展现内饰，只有简笔勾勒的座位，主角坐在车上，旁边只有一个方框是窗，窗上雾蒙蒙的，隐约可见外面的世界。这里没有背景音乐，只有白噪音。
	2. **初见这个世界**：花园，清泉，流水，小屋，但是仍然笼罩在雾气之中，营造那种阴郁的整体的氛围。这里的氛围会有点像山中好长日初见彩特琳德登上空之神殿时剧情的色彩，但是需要仔细斟酌表达的形式。这里是初见这个世界，游戏上可以介绍基础设定，然后可以有一些音乐小妖精作为初始的怪物。这里的背景可以是肖邦夜曲，但是也许需要做一些改编。
	3. **剧情发展**：黑森林，原始，阴森，有些令人恐惧。可以自己写点中古调式的曲子配乐。
	4. **逐渐进入高潮**：湖中尖塔，月光，月光会不断地变化，对应游戏内部战斗环境变化。这里的怪物暂且还是音乐相关的妖精。月光变换，对应战斗环境的调式变换。这里可以引入一些欧洲的神话，月亮在欧洲神话中常有与音乐有关的意向。场景有点水仙十字格式塔和深境螺旋的感觉。背景可以放月光奏鸣曲第一乐章。
	5. **塔尖**：第一个 boss 战，战斗曲可以在李斯特钟的基础上改，boss 本身也可以加入一些钟表元素。战斗结束后月光重新出现，但是整个场景只有朦胧的月光和被雾笼罩的夜空。主角将在这里完成一次世界观的展开：世界是倒悬的，而水面才是真实的星空。主角从塔上跳下，进入世界的反面。这里也是剧情的一个高点，象征从灰暗阴沉的物理世界走进纠结矛盾的内心世界。
	6. **倒置的塔**：向下离开尖塔，这时候内部景观需要有很大的变化。妖精消失，怪物变成调律不齐的乐器，背景音乐开始变得紧张，比如从月光一变成月光三。
	7. **来时路的重放**：倒着走一遍来时的路，但是怪物和环境都会有一些变化，整个氛围逐渐变得紧张和纠结，也可以增加一些高难的战斗。这里的战斗和探索也许可以放一些无调性的现代主义的音乐。
	8. **废弃车站**：第二个 boss 战，也是终局之战。可以继续参考山中好长日和水仙十字的写法。这一战的敌人象征内心的病魔和抑郁。战斗结束乌云散去，极光亮起。
	9. **结尾**：伴随拉二的音乐，列车启程。天空逐渐亮起，然后黎明到来，拨云见日。

## 3. 战斗系统 (Combat System)

核心机制命名为“和声共鸣 (Harmony Resonance)”。第一版不做复杂乐理模拟，也不做大调/小调互相克制，而是把战斗压缩成“音符标记 + 和弦反应 + 节奏结算”的回合制组合玩法。

### 3.1 战斗核心目标

玩家每轮行动的核心决策：

1. 用角色技能给敌人或己方挂上音符。
2. 凑出特定音程或和弦。
3. 用打击乐或特定技能触发共鸣。
4. 根据当前调性、节奏和敌人状态结算效果。

系统目标不是考验玩家的乐理知识，而是让玩家能在视觉上理解“我正在组合旋律”，并愿意为了凑出一个和弦改变行动顺序。

### 3.2 基础战斗资源

| 资源 | 作用 |
|------|------|
| **HP** | 常规生命值。 |
| **Tempo 节奏值** | 影响行动顺序、加速、延迟和额外行动。 |
| **Tone 音符标记** | 挂在单位身上的 C/D/E/F/G/A/B。 |
| **Resonance 共鸣槽** | 用于释放大招、改变场地调性或触发强力和弦效果。 |

第一版可以暂时不设计 MP。技能消耗如果太早引入，会让战斗验证目标变得不清晰。

### 3.3 音符标记 (Tone Marks)

角色技能会给目标挂上“音符标记”。第一版只使用自然音：

```text
C D E F G A B
```

每个单位身上最多保留 3 个音符。超过上限时，新音符顶掉最早的音符。

```text
敌人当前音符：[C, E]
长笛追加 G
敌人当前音符：[C, E, G]
形成 C-E-G，进入可触发的大三和弦状态
```

音符上限的意义：

- 降低玩家记忆负担。
- 避免状态无限堆叠。
- 让“补音”和“覆盖错误音符”成为清晰的战术选择。

### 3.4 和声共鸣 (Harmony Resonance)

不同音符组合会形成不同共鸣。第一版只保留 5 类，优先验证可玩性。

| 组合 | 名称 | 效果定位 |
|------|------|----------|
| C-E-G / F-A-C / G-B-D | **大三和弦** | 爆发伤害 + 己方攻击提升。 |
| A-C-E / D-F-A / E-G-B | **小三和弦** | 持续伤害 + 降低敌人 Tempo。 |
| 相邻音，例如 C-D、E-F | **不协和音** | 真实伤害 + 己方当前角色受到反噬。 |
| 相同音，例如 C-C | **齐奏** | 强化同音技能效果，可用于叠伤、治疗或护盾。 |
| C-G、D-A 等五度关系 | **完全五度** | 稳定、护盾、清除负面状态。 |

玩家不需要直接理解所有乐理定义，UI 上可以把它们表现为不同形状和色彩：

```text
隔一个音一个音：和弦
挨在一起：不协和
完全相同：齐奏
稳定的远距离连接：五度
```

### 3.5 共鸣触发规则

采用半自动触发：

- **小反应自动触发**：齐奏、不协和音。
- **大反应手动触发**：大三和弦、小三和弦、完全五度。

手动触发通常由打击乐、钢琴或特定技能完成。这样既保留即时反馈，也让玩家可以规划一轮“铺音符 - 补音 - 引爆”的战术节奏。

示例：

```text
小提琴给敌人挂 C
小提琴下一次行动追加 E
长笛追加 G
敌人身上形成 C-E-G
鼓手使用“重拍”，触发大三和弦
结算：高额伤害 + 己方攻击提升 + 共鸣槽增长
```

### 3.6 调性系统 (Tonality)

调性不再做大调/小调克制关系，而是作为场地规则存在。它改变当前战斗鼓励的策略方向。

| 调性 | 战斗效果 |
|------|----------|
| **Major 大调** | 爆发、护盾、增益效果更强。 |
| **Minor 小调** | 持续伤害、削弱、Tempo 控制更强。 |
| **Dissonant 失调** | 真实伤害更强，但双方受到的反噬也更高。 |
| **Silent 静默** | 音符持续时间减少，共鸣槽增长变慢。 |

Boss、钢琴角色或场景机制可以改变调性。

```text
月光转暗：
当前调性从 Major 变为 Minor。
所有小三和弦伤害 +30%。
所有大三和弦效果降低到 70%。
```

这样“月光变化”“塔内外反转”“无调性现代主义音乐”等叙事和美术意象，都可以落到具体战斗规则上。

### 3.7 角色职能分配 (Classes)

根据乐器发声原理分为不同战斗职责：

1. **弦乐 (Strings)**：多段攻击、群体伤害。负责铺设基础音符标记（主输出/辅助输出）。
2. **管乐 (Winds)**：单体高穿透、高爆发。负责精准击破或触发关键共鸣（刺客/狙击手）。
3. **打击乐 (Percussion)**：不参与凑和弦，负责**节奏控制 (Tempo)**。
    - **机制**：引爆场上和弦、拉条/推条、破坏敌方护盾（节奏稳定性）。
4. **钢琴 (Piano - 乐器之王)**：
    - **机制**：全能位。可补全缺失的任何音符；拥有“黑白键”双形态切换；通过“场地魔法”改变全场调性。
5. **留声机 / 主角：** 记录，复播，未必有战斗机制。

第一版 Demo 只需要三个角色：

```text
小提琴：铺音符、群体标记
长笛：精准补音、单体爆发
鼓：触发共鸣、推进己方行动条、延迟敌方行动
```

### 3.8 行动与节奏 (Tempo)

推荐使用行动条，而不是传统“我方全员行动 - 敌方全员行动”的回合制。

```text
速度越高，下一次行动越快。
打击乐可以推进己方行动条，也可以打断敌方节奏。
```

示例技能：

```text
鼓手技能：重拍
- 对目标造成少量伤害
- 触发目标身上的可触发共鸣
- 若触发成功，使己方全体行动提前 10%
```

节奏系统的目标是让“音乐主题”影响战斗手感，而不是只停留在命名上。

### 3.9 第一版数值模型

原型阶段先使用粗略公式，优先验证玩家是否理解决策反馈。

```text
普通伤害 = 攻击力 × 技能倍率 × 防御修正
共鸣伤害 = 触发者攻击力 × 共鸣倍率 × 调性修正
不协和真伤 = 攻击力 × 0.8，己方当前角色受到 20% 反噬
```

建议初始倍率：

| 反应 | 倍率 | 附加效果 |
|------|------|----------|
| 大三和弦 | 2.0x | 己方攻击 +15%。 |
| 小三和弦 | 1.2x | 2 回合 DoT，敌人 Tempo -10%。 |
| 不协和音 | 0.8x 真伤 | 己方当前角色受到本次伤害 20% 的反噬。 |
| 齐奏 | 1.0x | 同音符技能效果增强。 |
| 完全五度 | 0.7x | 己方获得护盾或清除 1 个负面状态。 |

### 3.10 基础战斗示例

队伍：

```text
小提琴：给敌人挂 C 或 E
长笛：给敌人挂 G，单体伤害较高
鼓：触发共鸣，推进己方行动条
```

敌人：雾中妖精。

```text
小提琴行动：给敌人挂 C。
敌人行动：攻击我方。
小提琴二技能：追加 E。
长笛行动：追加 G。
敌人身上音符为 C-E-G，形成大三和弦。
鼓行动：使用“重拍”，触发辉光共鸣。
效果：造成高额伤害，己方攻击提升 1 回合，共鸣槽 +20。
```

错误或冒险路线：

```text
敌人身上已有 C。
长笛挂 D。
形成 C-D 不协和。
立刻造成少量真实伤害，但己方当前角色受到反噬。
```

这样可以让玩家快速理解：音符不是越多越好，组合关系和行动顺序本身就是战斗策略。

## 4. 角色获取机制 (Character Acquisition)

本作不适合采用抽卡或持续运营式角色获取。角色应当作为“失落声部”的具象化存在，通过主线和支线逐步加入队伍，让玩家在单机流程中自然体验到主要机制。

### 4.1 获取结构

采用“主线核心队伍 + 支线扩展角色 + 终章前补全入口”的结构。

```text
主线必得：小提琴、长笛、鼓
支线获得：大提琴、双簧管、竖琴、钢琴等机制扩展角色
终章前补全：回声大厅 / 旧录音室，允许玩家回收未完成角色支线
```

主线只保证玩家拿到最基础的三人组：

```text
小提琴：铺音符
长笛：精准补音
鼓：触发共鸣 / Tempo 控制
```

这三人覆盖最核心循环：

```text
挂音符 -> 补音 -> 触发共鸣
```

后续角色通过地图支线获得。每个角色支线都应当是一个“小型音乐寓言”，用于教学一种新机制，而不是简单的收集任务。

### 4.2 角色支线设计原则

每个角色支线只引入一个主要机制：

| 角色 | 支线主题 | 解锁机制 |
|------|----------|----------|
| **大提琴** | 黑森林里失声的低音 | 防御、护盾、低速高收益。 |
| **双簧管** | 雾中迷路的信使 | 改写敌人音符、精准控制。 |
| **竖琴** | 湖边断弦的回声 | 治疗、净化、完全五度。 |
| **钢琴** | 尖塔中的黑白门 | 补全任意音符、切换调性。 |
| **留声机主角强化** | 废弃车站的旧录音 | 复播上一次共鸣。 |

支线结构：

```text
角色自身存在一个音乐上的缺损
地图或敌人围绕这个缺损设计
玩家通过战斗理解该角色的机制
支线 Boss 将该机制作为解法
通关后角色加入，缺损被补全
```

示例：竖琴支线

```text
湖边有一把断弦竖琴。
区域敌人会持续施加负面状态。
玩家需要利用 C-G、D-A 等完全五度来稳定旋律、清除噪音。
支线 Boss 是“缠结的弦”。
通关后竖琴加入，技能核心是完全五度、净化和护盾。
```

### 4.3 地图分布

主线不长，因此不应在主线中密集塞入大量角色。更适合采用“线性主线 + 可选回路”的地图结构。

```text
雪国列车
  ↓
雾中花园
  ├─ 核心队伍机制教学
  ↓
黑森林
  ├─ 支线：大提琴
  ├─ 支线：双簧管
  ↓
湖中尖塔
  ├─ 支线：竖琴
  ├─ 支线：钢琴
  ↓
倒置世界
  ├─ 支线：留声机强化
  └─ 高难回忆战
  ↓
废弃车站
```

关键机制角色可以“主线遇见，支线入队”。例如钢琴角色可以在湖中尖塔主线中出现并改变场地调性，但真正加入队伍需要完成黑白门支线。

### 4.4 防止玩家错过机制

为了让玩家尽量体验到主要机制，需要三层保障：

1. **核心机制主线必得**：小提琴、长笛、鼓不可错过。
2. **关键机制强提示**：钢琴、竖琴等关键角色支线在主线路径上明显出现。
3. **终章前补全入口**：废弃车站前开放“回声大厅 / 旧录音室”，允许玩家重新进入未完成的角色支线。

奖励不必全部是完整角色，可以分为三类：

```text
完整角色：改变队伍构成
角色记忆：强化已有角色技能
乐章碎片：解锁共鸣、被动或调性规则
```

这样探索奖励不会完全被角色数量绑定，也能避免短流程里角色过度膨胀。

## 5. 美术与探索设计 (Art & Exploration)

战斗和剧情对话基本确定为 Live2D + 字幕 / 卡牌回合制。探索部分建议采用固定视角的 2.5D / 伪 3D 舞台式探索，而不是完整横板动作或纯房间切换。

### 5.1 三种方案对比

| 方案 | 优点 | 风险 |
|------|------|------|
| **2D 横板探索** | 适合上下移动、爬塔、平台结构，制作逻辑直观。 | 容易让玩家期待动作游戏手感；信息承载量有限；如果没有跳跃、攻击、平台挑战，会显得空。 |
| **2D 房间切换** | 成本低，地图结构清晰，适合事件和战斗触发。 | 表达“塔外月光变化”“倒悬世界”“空间透视”的能力较弱，氛围容易变成菜单式关卡。 |
| **固定视角 2.5D / 伪 3D** | 能表现透视、塔楼、倒置、月光、舞台感；与神秘学和古典乐气质更贴合。 | 需要更强的场景构图能力；如果做成完整《纪念碑谷》式机关解谜，制作成本会上升。 |

推荐方向：

```text
固定视角 2.5D 舞台式探索
```

它不需要做成真正的 3D 动作游戏，也不需要完整复刻《纪念碑谷》的空间谜题。核心是用固定镜头、透视构图、层次遮挡和少量空间机关来表达“世界是一座被音乐和记忆折叠的舞台”。

### 5.2 探索表现形态

推荐做成“章节节点 + 局部可走场景”的结构。

```text
大地图 / 章节推进：线性主线 + 支线回路
局部探索：固定镜头场景，玩家在场景内移动、调查、进门、上下楼
关键场景：用 2.5D 透视和镜头构图强化叙事
```

典型场景表达：

- **雪国列车**：近似 2D 剧场横切面，窗外只有雨雾和白噪音。
- **雾中花园**：固定视角庭院，前景花枝、中景小路、远景小屋和泉水。
- **黑森林**：多层树影遮挡，路径像乐谱线一样交叉。
- **湖中尖塔**：塔楼外侧有固定月光窗口，玩家上下移动时月色和调性发生变化。
- **倒置的塔**：相同场景反向重走，但重力、楼梯、窗外星空和敌人配置全部变化。

### 5.3 塔楼与上下移动的处理

爬塔不一定要依赖横板平台跳跃。可以用固定视角 2.5D 场景表达“上升”和“下降”：

```text
螺旋楼梯：角色沿固定路径上行，背景窗外月光逐层变化
升降平台：场景整体纵向移动，暗示进入塔的更高层
门与窗：门负责场景切换，窗负责展示外部月色和世界状态
倒置桥段：同一组楼梯在倒置世界中变成向下通路
```

这样保留爬塔叙事，又不需要把游戏做成动作平台跳跃。

### 5.4 美术技术方案

推荐技术实现：

```text
Unity 2D + 正交相机
2D 手绘场景图层放置在 3D 空间中
前景 / 中景 / 远景分层视差
少量 3D 几何体辅助楼梯、门、塔身透视
Live2D 用于剧情和战斗，探索中使用简化 2D 小人或剪影
```

探索角色不一定需要高成本 Live2D，可以使用：

- 小比例 2D 行走角色。
- 剪影化主角。
- 简化骨骼动画。

高质量 Live2D 资源集中用于：

- 剧情对话。
- 战斗角色。
- Boss 登场。
- 关键支线角色初见。

### 5.5 视觉基调

美术关键词：

```text
欧洲中世纪 / 近代
神秘学
古典乐
雪、雾、月光、极光
舞台布景感
倒悬空间
乐谱线与建筑线条融合
```

需要避免的方向：

- 纯动作平台跳跃感过强。
- 房间切换过于功能化，缺少空间诗意。
- 伪 3D 解谜过重，喧宾夺主。
- 场景过度华丽，导致 Live2D 角色和卡牌 UI 被淹没。

最终目标是让探索承担“氛围、空间、支线入口、轻量谜题”的职责，而不是承担复杂动作系统。

## 6. 技术选型 (Technical Specs)

- **开发引擎**：**Unity**。
    
    - **理由**：对 Live2D/Spine 的 2D 骨骼动画支持最成熟，适合做高质量立绘交互；UI 系统（UGUI/UI Toolkit）适合重度 UI 的卡牌 RPG。
        
- **音频处理**：推荐引入 **FMOD** 或 **Wwise** 中间件，用于处理复杂的动态音轨叠层和实时调性切换。
    
- **技术挑战与对策**：
    
    - **UI 性能**：采用 Canvas 动静分离。
        
    - **资源管理**：使用 Addressables 实现音频与美术资源的异步加载。
        
    - **内存优化**：针对频繁生成的音符/特效使用对象池 (Object Pool)。

## 7. 开发路线 (Roadmap)

### Phase 0：纸面规则与数值沙盘

目标：先在表格或简单脚本里验证“音符组合是否有决策价值”。

- 明确 7 个自然音、5 类共鸣、2-4 种调性。
- 设计 3 名初始角色：小提琴、长笛、鼓。
- 设计 2 种普通敌人和 1 个小 Boss。
- 明确角色获取结构：主线核心队伍、支线扩展角色、终章前补全入口。
- 用表格模拟 5-10 场战斗，检查是否存在无脑最优解。

### Phase 1：战斗原型 Prototype

目标：优先做战斗，同时验证探索表现的最小技术方案。

- 实现行动条与 Tempo。
- 实现音符标记上限、覆盖规则和 UI 显示。
- 实现齐奏、不协和、大三和弦、小三和弦、完全五度。
- 实现手动触发共鸣的打击乐角色。
- 做一个可重复挑战的测试战斗场景。
- 做一个固定视角 2.5D 探索灰盒，验证移动、进门、上下楼和视差层。

验收标准：

- 玩家能在 3 分钟内理解“挂音符 - 补音 - 触发共鸣”。
- 玩家会主动为了凑和弦改变技能选择。
- 不协和音既有收益，也有明确风险。

### Phase 2：15 分钟 Demo

目标：把战斗原型放进一个最小叙事体验。

- 场景：雪国列车 -> 雾中花园。
- 队伍：小提琴、长笛、鼓。
- 敌人：音乐妖精、失谐乐器。
- Boss：钟表/铃声主题敌人。
- 探索：固定视角 2.5D 场景，包含至少 1 个支线入口。
- 音乐：先用 Unity AudioMixer 做简单分层，不急着接 FMOD/Wwise。
- 剧情：完成一次“世界停在休止符上”的开场表达。

验收标准：

- 单次流程 10-15 分钟。
- 至少出现一次调性变化。
- Boss 战要求玩家稳定使用和声共鸣，而不是只靠普通攻击。

### Phase 3：垂直切片 Vertical Slice

目标：证明项目的完整体验形态可成立。

- 完成“雪国列车 - 花园 - 黑森林 - 湖中尖塔”中的一个完整章节。
- 加入钢琴角色或钢琴 Boss 机制，用于验证调性切换。
- 完成 1-2 条角色支线，验证“角色机制教学 + 入队”的结构。
- 加入基础养成：技能升级或共鸣强化，二选一即可。
- 确立角色美术方向：保留乐器特征的拟人化设计。
- 确立探索美术方向：固定视角 2.5D 舞台式场景。
- 初步建立动态音乐规则：战斗层、调性层、Boss 阶段层。

验收标准：

- 有 30-45 分钟可游玩内容。
- 战斗、剧情、美术、音乐能形成统一气质。
- 玩家能说清楚本作和普通卡牌 RPG 的差异。

### Phase 4：内容扩展

目标：在垂直切片成立后，再扩大系统。

- 扩展角色职业：弦乐、管乐、打击乐、钢琴、留声机主角。
- 扩展角色支线：每个支线绑定一个新机制或一个角色记忆。
- 扩展调性：Major、Minor、Dissonant、Silent。
- 增加更多 Boss 的场地机制。
- 引入角色构筑，但避免过早做复杂装备系统。
- 评估是否引入 FMOD 或 Wwise 处理动态音轨。

### Phase 5：打磨与发布准备

目标：把体验从“概念成立”打磨到“可以公开展示”。

- UI 可读性和音符提示优化。
- 战斗节奏、数值曲线和难度曲线调整。
- 音乐、美术、演出资源统一风格。
- 性能优化：对象池、Addressables、Canvas 动静分离。
- 准备试玩版、宣传素材和反馈收集表。
