### 关于 RAG 的部分

这部分大概可以做成一个 MCP 服务的类似物，或者直接暴露 HTTP 接口，从而既可以给人使用也可以暴露 MCP 给 Agent 使用。

>[!important] 关于时空索引的数据库
>在和 Gemini 的讨论中提到了一个建立时空索引的思路，这似乎是有道理的。这个也可以作为一个创新点，但是对于空间的工作会有些小困难。

### 关于 Agent 的部分

考虑到目录已经是给出了的，因此不太需要一个目录的 Agent 。

这里可以给出三个 Agent：

- Researcher Agent：负责查找数据库，给出大纲。它需要做多轮工作，类似于追问。
- Writer Agent：从 Researcher 给出的内容撰写一段文本。
- Judger Agent：用于评判 Writer Agent 的内容中的真实性。如果不需要严谨的真实性，这一点似乎可以忽略。
