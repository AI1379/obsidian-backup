# 透明容器液体残留检测 — 文献调研报告

> **任务背景**：数学建模竞赛题 — 判断透明塑料饮料瓶内是否有残留液体，需构建数据集（≥100 张）、建立检测模型、并尝试扩展至瓶体定位和液体含量分级。
> **技术路线**：YOLO 目标检测 + 传统 CV（边缘检测/Hough 变换等）+ AI 辅助打标
> **调研日期**：2026-05-08

---

## 一、任务拆解与技术需求

| 子任务 | 核心技术 | 可参考方向 |
|--------|---------|-----------|
| 数据集构建 | 自拍摄 + 公开数据 +AI 辅助打标 | Trans10K, LCDTC, SAM 自动标注 |

### 2.5 传统 CV 方法（Hough 变换/边缘检测/颜色分析）

这些方法虽然不在 arXiv 上大量发表，但在工程实践中非常有效：

#### [E1] Canny 边缘检测 + Hough 直线变换
- **用途**：检测液面水平线（液体在瓶中形成近似水平线）
- **适用场景**：瓶子竖立或近似竖立时，液面是一条清晰的水平线
- **局限**：瓶子横放/倾斜时液面不是直线，需用其他方法

#### [E2] 颜色空间分析（HSV/Lab）
- **用途**：将 ROI 转到 HSV 空间，检测非背景色的液体颜色
- **方法**：对瓶体区域做颜色直方图分析，空瓶 vs 有液瓶在饱和度/亮度分布上有明显差异
- **适用场景**：有色液体（茶、果汁、可乐等）

#### [E3] 纹理分析（LBP/Gabor）
- **用途**：液体区域与空气区域的纹理不同
- **方法**：对瓶体上下区域分别计算 LBP 特征，比较差异

#### [E4] 光流/折射分析
- **用途**：利用液体的折射特性，液体区域和空气区域的光学表现不同
- **适用场景**：视频序列分析

## 三、推荐技术方案

基于文献调研，推荐如下技术路线：

### 阶段一：数据集构建
1. **自拍摄**：收集不同品牌、姿态、液体类型的饮料瓶照片（建议 150-200 张）
2. **公开数据补充**：
   - LCDTC 数据集中的饮料瓶子集
   - Trans10K 中的瓶子类
   - UAV-BD 中的相关图片
3. **AI 辅助打标**：
   - Grounding DINO + SAM 自动生成瓶体 mask
   - 人工修正 + 液体标签标注
   - 工具：LabelMe / CVAT / Roboflow

### 阶段二：核心模型
1. **瓶体定位**：YOLOv8/v11（轻量、成熟、好部署）
2. **液体判断**（推荐混合方案）：
   - **方法 A（纯 DL）**：YOLO 检测瓶体 → 添加液体分类头（类似 LCD-YOLOX 的额外 head）
   - **方法 B（DL+CV 混合，推荐！）**：
     - YOLO 定位瓶体区域 → ROI 裁剪
     - 传统 CV 分析 ROI：
       - Hough 变换检测液面线
       - HSV 颜色分析检测液体颜色区域
       - 上下区域纹理对比
     - 特征融合 → SVM/随机森林/简单 MLP 做最终判断
3. **液体含量分级**：在分类头输出 3 类（少量/中等/较多）或回归

### 阶段三：评估与扩展
- **指标**：Accuracy, Precision, Recall, F1, mAP（如有定位）
- **消融实验**：对比纯 DL vs DL+CV 混合方案
- **泛化测试**：在未见过的瓶型和液体类型上测试

## 📄 Phys-Liquid: 透明可变形液体的 3D 几何与体积估计

**AAAI-26 Oral** · 华中科技大学 + NYU Abu Dhabi + BOKU · Ke Ma et al.

---

### 🧪 数据集：Phys-Liquid

| 维度 | 详情 |
|:---|:---|
| **规模** | **97,200 张仿真图像** + 8,100 个标注 3D 网格 (OBJ) |
| **容器** | **20 种**实验室容器（立方体、圆柱、锥形瓶、球瓶、复合形状） |
| **场景** | 5 个实验室场景（含实验台、设备） |
| **光照** | 8 种室内光照条件（强度/方向/颜色/混合变化） |
| **液体颜色** | 5 种（无色、紫、红、橙、黄） |
| **旋转模式** | 6 种轴组合（绕 X/Y/Z），0°~80°，每序列 81 帧 |
| **仿真引擎** | Blender + Mantaflow，基于 **Navier-Stokes 方程** 物理仿真 |
| **视角** | 6 个正交相机（前后左右上下） |
| **元数据** | 每张图附带液体掩码、3D 网格、物理参数、旋转角度、真实尺寸 |

每张图都配了 RGB + 液体 mask + 3D 网格 + 物理渲染元数据。

**真实性验证**：拿真实实验拍摄对照，液体表面角度误差很小（论文图 5 红线/绿线对齐）。

---

### 🔧 四阶段重建管线

```
输入单图 → ①分割 → ②多视图掩码 → ③3D重建 → ④真实缩放
```

1. **液体分割**：**YOLO-world** 检测液体边界框 → **SAM2** 精细分割（显式排除 " 瓶子 " 类，只保留 " 液体 "/" 有色液体 "）

2. **多视图掩码生成**：**CRM 扩散模型** 在 Phys-Liquid 上微调（6 个正交视图），从单视图生成其余 5 个视角的液体掩码 + 规范坐标图 (CCM)
   - 微调后 IoU：74.38% → **90.05%**

3. **3D 网格重建**：U-Net 编码多视图掩码 → 三平面 (triplane) 表示 → MLP 解码 → Flexicubes 生成纹理 3D 网格

4. **真实世界缩放**：多视图 ViT → Transformer Encoder → MLP 回归缩放因子 s，将重建网格映射到真实尺寸

### 📊 实验结果

**vs 专用基线 (Eppel et al. TransProteus)**：

| 指标 | Eppel et al. | Phys-Liquid (本方法) |
|:---|:---:|:---:|
| RMSE | 0.0842 | **0.0192** ↓77% |
| Chamfer Distance | 0.0412 | **0.0079** ↓81% |
| Volume IoU | 0.1216 | **0.4748** ↑290% |
| F-Score | 30.91% | **75.38%** |

## 📄 Trans2Seg: Segmenting Transparent Object in the Wild with Transformer

**arXiv 2021** · 香港大学 + 商汤科技 + 南京大学 · Enze Xie et al.

---

### 🏷️ 数据集：Trans10K-v2

这是透明物体分割领域最著名的大规模数据集之一：

| 维度 | 详情 |
|:---|:---|
| **规模** | **10,428 张真实场景图片**（5000 训练 / 1000 验证 / 4428 测试） |
| **类别** | **11 个细粒度类别**，分为两大类 |
| **透明物品** (Things) | cup 杯子、bottle 瓶子、jar 罐子、bowl 碗、eyeglass 眼镜 |
| **透明结构** (Stuff) | window 窗户、shelf 货架、box 储物盒、freezer 冰柜、glass wall 玻璃墙、glass door 玻璃门 |
| **场景** | 13 种室内场景（桌面、商场、卧室、厨房、办公室等） |
| **标注** | 高精度手工标注 mask，标注员每边界≥100 个点 |

每张图都配有场景标签，详细标注了透明物体位置。**"bottle"（瓶子）是 11 个类别之一**，这对你来说非常实用！

---

### 🏗️ Trans2Seg 架构

```
输入图像 → CNN 骨干网 → Transformer Encoder → Transformer Decoder → Conv Head → 分割 Mask
```

关键创新在于 **Transformer Decoder 的设计**：

1. **Transformer Encoder**：将 CNN 提取的特征展平 + 位置编码，用多头自注意力获取**全局感受野**（这对透明物体特别关键——透明物体常和背景纹理相似）

2. **Transformer Decoder**：设计了一组**可学习的类别原型 (learnable class prototypes)** 作为 query，每个原型对应一个语义类别，通过注意力机制在特征图中 " 查找 " 对应像素
   - 核心思想：把语义分割建模成 " 字典查找 " 问题
   - query = 类别原型嵌入，key = 编码器特征图，输出 = 注意力图

3. **Small Conv Head**：注意力图 + CNN 浅层特征 Res2 融合 → 最终预测

这个设计比 SETR（只用 CNN decoder）和 DETR（query 代表目标实例而非类别）都更适配语义分割。

---

### 📊 实验结果

**Ablation（逐步加模块）**：

| 配置 | mIoU |
|:---|:---:|
| 纯 FCN (baseline) | 62.7% |
| + Transformer Encoder | 68.8% (+6.1) |
| + Transformer Decoder (完整 Trans2Seg) | **72.1%** (+3.3) |

**vs 20+ CNN 分割方法（Trans10K-v2 测试集）**：

| 方法 | FLOPs | mIoU |
|:---|:---:|:---:|
| FCN | 42.2G | 63.0% |
| DeepLabV3+ | 38.0G | 68.9% |
| PSPNet | 187G | 68.2% |
| TransLab (之前 SOTA) | 61.3G | 69.0% |
| **Trans2Seg** | **49.0G** | **72.2%** ✅ |

在 **bottle**（瓶子）类别上，Trans2Seg 比之前 SOTA 高出 **10%** IoU（全局感受野对小物体特别有效）。

在 **ADE20K** 通用数据集上也验证了迁移能力（39.7% mIoU）。

## 🧩 具体结合方案

**方案 A：实用路线（YOLO + Trans2Seg 辅助）**

```
输入图像
  ├─ YOLOv8 检测瓶体位置 (LCDTC 思路)
  ├─ 裁剪瓶体区域
  ├─ Trans2Seg 精细分割瓶子/液体轮廓
  └─ 掩码面积比 → 液位分级 (5级或更细)
```

- 轻量、可部署
- 用 Trans10K-v2 的 bottle 标注微调分割头
- 液体残留 binary 判断用面积阈值

**方案 B：精度路线（Phys-Liquid 体积估计）**

```
输入图像
  ├─ Trans2Seg 分割瓶体 + 液体区域
  ├─ Phys-Liquid 管线：
  │   ├─ SAM2 精细分割液体 mask
  │   ├─ 多视图掩码生成（CRM 扩散模型）
  │   └─ 3D 重建 → 体积估计
  └─ 连续体积值 → 映射到含量等级
```

- 精度最高，能从单张 2D 图估计 3D 体积
- 但管线重、需要 GPU，可能超出赛题时间

**方案 C：混合路线（推荐）**

```
阶段一（快速定位）：YOLOv8 检测瓶体 + LCDTC 5级粗分类
阶段二（精细分析）：对"有残留"样本用 Trans2Seg 分割
阶段三（含量估计）：基于分割 mask 的面积/形状 + 瓶体几何先验 → 回归连续液位
```

- 取 LCDTC 的轻量级检测思路
- 取 Trans2Seg 的精细分割能力（尤其处理透明边界）
- 用 Phys-Liquid 的仿真数据做数据增强或预训练
- **不照搬 3D 重建管线**，而是简化：2D mask → 面积比例 → 体积估计公式（利用瓶体几何形状已知的先验）

---

### 💡 最关键的点

LCDTC 只做了 **5 级离散分类**，Phys-Liquid 能做 **连续体积估计**。如果你们赛题的任务 3 要求 " 含量分级 " 而不是简单的有/无判断，把 Phys-Liquid 的体积回归思路简化后套到 2D mask 上，是一个很好的亮点。
