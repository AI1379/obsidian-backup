# GAM: Hierarchical Graph-based Agentic Memory for LLM Agents

## 论文信息
- **arXiv**: 2604.12285
- **标签**: Agent / Memory / 知识图谱 / LLM
- **日期**: 2026-04-15 (arXiv 精选)

---

## 核心思想

提出一种**分层图结构**的记忆框架，模拟人类记忆系统（工作记忆 → 情景记忆 → 语义记忆），为 LLM Agent 提供更智能的记忆能力。

## 与传统记忆框架的对比

| 特性 | 传统 RAG / 向量库 | GAM (图分层记忆) |
|---|---|---|
| **存储结构** | 扁平向量数据库 | **分层图结构** |
| **检索方式** | 相似度检索 (ANN) | **图遍历** + 语义检索 |
| **关系建模** | 无 | **显式建模实体关系** |
| **层次化** | 无 | **多层次** (工作→情景→语义) |
| **动态更新** | 简单追加 | **图结构动态剪枝** |

## 与 GraphRAG 的关系

你说得对！GAM 和 GraphRAG 确实有很强的关联：

### 共同点
1. **图结构**：都用图来组织知识，显式建模实体间关系
2. **层次化**：都意识到单层存储不够，需要分层
3. **语义感知**：不仅仅是关键词匹配，而是理解语义关系

### GAM 的独特之处
1. **更细粒度的分层**：
   - 工作记忆（Working Memory）：当前对话上下文
   - 情景记忆（Episodic Memory）：近期交互历史
   - 语义记忆（Semantic Memory）：长期知识/事实

2. **Agent 导向**：
   - 专门为 Agent 任务设计（规划、推理、工具调用）
   - 记忆会动态影响 Agent 的决策

3. **动态图结构**：
   - 根据任务需要动态扩展/剪枝图结构
   - 不只是静态存储

### GraphRAG 的特点
- 侧重于 **RAG 场景**（检索增强生成）
- 用图索引增强向量检索
- 更关注知识问答而非 Agent 决策

## 技术细节

### 分层设计
```
┌─────────────────────────────────────┐
│        Semantic Memory             │  (长期知识、事实)
│   (知识图谱、概念关系)                │
├─────────────────────────────────────┤
│       Episodic Memory              │  (历史交互)
│   (对话历史、事件序列)                │
├─────────────────────────────────────┤
│        Working Memory             │  (当前上下文)
│   (当前prompt、临时变量)              │
└─────────────────────────────────────┘
```

### 检索策略
- 根据任务类型从不同层级检索
- 图遍历 + 向量检索混合
- 动态权重调整

## 个人评价

GAM 可以说是 **GraphRAG 的演进版本**，从"增强检索"到"增强 Agent 决策"。如果你对 GraphRAG 感兴趣，这篇论文值得一读。
